1.1.        CAF Command Structure

Each command contains the commands target and 0 or more parameters. The commands target is made of an app ID, a module ID and a command ID:

        App ID A set of up to 4 characters, including upper-case letters and numbers, that is compacted into a 24-bit value, and stored at the upper 24 bits of a 32-bit variable. In Tcl the compacting is done automatically for the first argument of golem::sendcmd, and can also be done with golem::id2hex. In C the compacting is done with the macro ID().

For example, in Tcl, the following produces the value 0xB32D2300:

golem::id2hex CTRL

For example, in C, the following produces the value 0xB32D2300:

ID('C','T','R','L')

The app ID APP_ID_BROADCAST (hexadecimal 0xFFFFFF00) is interpreted as broadcast to all existing apps.

        Module ID A positive integer that enumerates modules within a specific instance of the Module App.

The enumeration is an arbitrary number in the range from 1 to 254, including.

The module ID MODULE_ID_APP (value 0) is interpreted as targeting the command to the app itself.

The module ID MODULE_ID_BROADCAST (value 255, i.e. hexadecimal 0xFF) is interpreted as broadcast to all existing modules within a specific instance of the Module App.

        app_module_id A 32-bit variable holding an app ID at the upper 24 bits and a module ID at the lower 8 bits. In Tcl this is done automatically for the first and second arguments of golem::sendcmd in case the module ID is present, and can also be done with golem::id2hex. In C the or operator | is used to combine the IDs.

For example, in Tcl, the following produces the value 0xAE2B3014:

golem::id2hex PLBK 20

For example, in C, the following produces the value 0xAE2B3014:

ID('P','L','B','K') | 20

         command_id Identifies the command to the recipient. There are two types of commands:

SET command Bit 8 in command_id is clear The sender is writing data or signaling the recipient, and the payload of the reply is therefore assumed to be empty.

GET command Bit 8 in command_id is set The sender is retrieving data from the recipient and both incoming and outgoing commands can contain a data payload.

A command ID can be specified numerically or as a variable or as a C language pre-processor definition. For example, the command ID can be written as 0x0007 or MODULE_CMD_SET_ACTIVE to set a certain module into active or bypass mode and as 0x0107 or MODULE_CMD_GET_ACTIVE to query a certain module if it is in active or bypass mode.

The command can hold parameter values that are passed according to the specified target:

        data An optional set of 32-bit values, which is the payload of the command. Its size varies among commands and can be up to 13. Parameter values can be specified numerically or as variables or in some cases as C language pre-processor definitions. For example, it is possible to use 0 or MODULE_BYPASSED for setting a module to bypass mode and to use 1 or MODULE_ACTIVE for setting a module to active mode.

        num_32b_words The number of 32-bit elements in the commands payload.

SET command When the command is sent, num_32b_words holds the size of the data payload in 32-bit words. In the reply, it holds a non-positive return code (see below).

GET command When the command is sent, num_32b_words holds the size of the memory area thats available to store the recipients data in 32-bit words. In the reply, num_32b_words holds the size of the data stored in the payload in 32-bit words or a negative return code in case of a failure (see below).

The implementation of the command interface in C code uses the following definitions to allow a data structure with a variable size:

#define COMMAND_OF_SIZE(n) \

struct { \

signed int num_32b_words:16;\

unsigned int command_id:15; \

unsigned int reply:1; \

unsigned int app_module_id; \

unsigned int data[n] ; \

}

 

// The maximum number of 32-bit data elements that a command can contain

#define MAX_COMMAND_SIZE 13

 

typedef COMMAND_OF_SIZE(MAX_COMMAND_SIZE) Command ;

The type Command* may be used for referring to command structures of arbitrary sizes, for example:

COMMAND_OF_SIZE(5) cmd5;

Command *cmd = (Command *)&cmd5;

The C data-structure contains a reply flag:

        reply Identifies the direction of the command. If the bit is clear, the command is a request from the host to the device. The sender clears the bit before sending the command. If the bit is set, the command is a reply from the device to the host. The recipient sets the bit after it completed processing the command.

1.2.        Return Codes

The returned value of a command is one of the following:

        MODULE_RC_SUCCESS (enumerated as 0) if the command has completed successfully and there is no data to return.

        A positive value indicating the number of 32-bit elements that are returned through the data field if the command has completed successfully and there is data to return.

        An error code as listed in table 5-1 (enumerated as negative values) if there has been some error condition at the Extension Layer or an error code as listed in table 5-2 (negative of standard errno codes) if there has been some error condition at the SOS level.
